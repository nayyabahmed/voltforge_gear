# Contributing

Thanks for helping build the VoltForge Gear handbook. Two things matter more
than anything else here:

1. **The reader is 11 years old.** Clarity fixes and better analogies are as
   valuable as technical corrections.
2. **Consistency beats brilliance.** A chapter that follows the conventions
   is easier to trust than a clever one that does not.

## Before you write

- Read [the style guide](style-guides-principles/STYLE-GUIDE.md) - voice,
  chapter template, vocabulary rules, visuals policy and the character-set
  rule (ASCII only, except inside mermaid blocks and real engineering
  notation).
- Read [the guiding principles](style-guides-principles/GUIDING-PRINCIPLES.md) -
  the project's engineering philosophy. Chapters must not contradict it.
- Check [SUMMARY.md](SUMMARY.md) for the agreed chapter plan and numbering.

## Writing or changing a chapter

- One file = one complete chapter, in the correct `Part-N-Name/` folder.
- Follow the chapter template in the style guide, including the publishing
  checklist (glossary entries, SUMMARY.md status, verified links,
  cross-references to the real neighbouring chapters).
- Verify technical facts against a reliable source before stating them, and
  be strictest with safety content (batteries, hot tools, cutting).
- New documents start at v0.1 Draft and mature through the lifecycle in
  [VERSIONING.md](VERSIONING.md).

## Pull requests

- One chapter (or one focused fix) per pull request.
- Say in the PR description what an 11-year-old gains from the change.
- Expect review feedback about reading level - it is the point, not a nitpick.

## Found a problem but can't fix it?

Open an issue with the chapter, the line or section, and what confused you.
"I didn't understand this" is a complete, useful bug report for this project.
